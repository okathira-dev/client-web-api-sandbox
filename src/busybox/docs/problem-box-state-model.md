# 問題箱の形状と再挑戦モデル

## 単一の箱形状

登録された全ステージの全問題箱は、単一の `GiftBox` コンポーネントで表現する。問題ごとに変えてよいのは箱の色と直下のヒントだけであり、次は変えない。

- DOM構造
- 箱本体、蓋、リボンの形と位置
- 縦横比と問題箱の寸法
- 開閉アニメーション
- フォーカス枠と操作領域

ステージ一覧の箱も同じコンポーネントを使う。一覧では一覧性を優先した等比縮小表示にしてよいが、問題箱同士は同じ寸法にする。APIごとの形状variantは作らない。

ヒントは箱の内部へ描かず、各箱の直下へ対応付ける。S-010ではマウスカーソル、指、ペンのSVGをそれぞれB01、B02、B03の直下へ置く。色やアイコンだけでクリア状態を示さず、リボンと蓋の形状を必ず使う。

## 永続履歴と今回の入場

問題箱の外見は、IndexedDBへ保存する永続履歴と、現在のステージ入場だけに存在する一時状態から導出する。

| 過去にクリア | 今回クリア | 箱の外見 |
| --- | --- | --- |
| いいえ | いいえ | リボン付きの閉じた箱 |
| はい | いいえ | リボンなしの閉じた箱 |
| 問わない | はい | 蓋が開いた箱 |

チェックマーク、疑問符、丸いステータス表示を問題のクリア表現には使わない。ステージを開くたび、過去にクリア済みの問題もリボンなしの閉じた箱から再挑戦する。今回条件を満たした時だけ蓋を開く。

## 入場の境界

一覧、別ステージ、設定、AboutからステージURLへ入った時、履歴で戻った時、直接URLを開いた時、再読込した時に新しい入場を始める。別タブへの切替、権限ダイアログ、通知、同じステージ内のフォーカス移動では入場を終えない。

今回開いた箱の集合はメモリだけに置き、IndexedDB、JSON書き出し、Driveへ保存・同期しない。`solve` は今回の箱を開いた後で既存のgrow-only進捗へ事実を記録する。過去にクリア済みでも今回の箱は必ず開き、永続側の初回 `solvedAt` は変更しない。

## ステージ一覧との違い

ステージ一覧は累積実績を表すため、従来どおり未着手をリボン付き、一部解決をリボンなしの閉箱、完全解決を開箱とする。各カードには説明文を置かず累積値を`x/n`で表示し、カード全面を1つのbutton操作領域にする。ステージ内ヘッダーのクリア問題数も累積実績を表示し、再入場して箱が閉じ直しても減らさない。たとえばS-010でB01を一度開いた後の再入場は、箱の外見が「B01はリボンなし閉箱、B02/B03はリボン付き閉箱」であっても、一覧とヘッダーはいずれも`1/3`相当を示す。

## アクセシビリティ

- 箱の状態はリボンと蓋で区別し、色だけへ依存しない。
- 各問題箱には「未クリア」「過去にクリア・今回未クリア」「今回クリア」の日英ラベルを付ける。
- 操作可能な箱だけをbuttonとし、外部イベントで開く箱を偽のbuttonにしない。
- `prefers-reduced-motion` では蓋の移動を瞬時にし、状態差は維持する。
- SVGヒントは装飾扱いとし、問題箱のラベルへ意味を含める。

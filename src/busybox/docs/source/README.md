# 添付資料の保存版

このディレクトリは、ユーザーから受領した企画書と調査メモを、後から意図と差分を追えるように保存する。

| 文書 | 位置づけ |
| --- | --- |
| [企画書ドラフト](./product-brief-draft.md) | プロダクト意図の入力資料 |
| [Deep Researchメモ](./deep-research-report.md) | Web APIギミックの未検証アイデア集 |

## 文書の優先順位

内容が矛盾する場合は、次の順に扱う。

1. `../decision-log.md` の確定事項
2. `../product-spec.md` と `../architecture-decisions.md`
3. 各台帳と実装計画
4. このディレクトリの添付資料

添付資料の文面、互換性、引用マーカーをそのまま実装根拠にしない。API採用前に、MDN、仕様策定元、ブラウザ公式情報、実機試作で再確認する。Deep Researchメモの各案はあくまでアイデアであり、採用、既存案への統合、保留、却下のいずれかを台帳へ記録してから実装する。

# Math.random 推測・検証ツール — 製品仕様

本書はウェブアプリとして**何ができるか**・**どう見せるか**を定義する。

## 1. 位置づけ

| 項目 | 内容 |
| --- | --- |
| 主用途 | 研究・検証（観測を増やしたときの絞り込みの進み方を調べる） |
| 副用途 | 教育（実装依存性と観測形式の影響の理解） |
| 非目的 | 系列種別の A/B 比較 UI、特定サービスへの攻撃手順、暗号学的乱数の品質評価 |

ユーザーが得る主な価値:

- **いまどれだけ内部状態が絞れているか**（候補数・信頼度）
- **まだどれだけ情報が足りないか**（残り不確実性 bit）
- **完全に 1 つに特定するにはあと何観測くらい必要か**（推定残り観測数）
- 観測を 1 件足すたびに、上記が**リアルタイム**で更新されること

「次の乱数の当たり外れ」は副次的な表示とする。

## 2. 用語

| 用語 | 定義 |
| --- | --- |
| 観測値 | 同一乱数源からの連続出力の 1 要素 |
| 観測系列 | 観測値の並び（順序保持・欠落なしが前提） |
| 生系列 | `Math.random()` の `[0, 1)` 浮動小数 |
| 変換系列 | `Math.floor(Math.random() * N)` の結果（`N` はユーザー指定） |
| 観測系列種別 | 生系列 **または** 変換系列のどちらか一方（同時には扱わない） |
| 厳密観測値 | 初版で扱う観測値。生系列は入力された 10 進文字列を JS `Number` に parse した結果を、観測された `Math.random()` 値そのものとして扱う |
| 利用アルゴリズム | 推論に用いる PRNG モデル（モデル一覧から 1 つ） |
| 生成源（デモのみ） | 系列を作る実装（モデル実装、または `Math.random()`） |
| 内部状態候補 | 観測系列と整合する内部状態の候補 |
| 完全特定 | 内部状態候補が 1 通りに定まること。次の出力だけが一意に見える状態とは区別する |
| 絞り込み状況 | 候補数・信頼度・推定残り観測数などをまとめた表示 |
| 残り不確実性 | 現在の候補空間の大きさを bit で表した目安。候補数 `C` に対して概ね `log2(C)` bit として表示する |

## 3. トップレベル構成

### 3.1 2 つのモード

| モード | 乱数生成 | 主な操作 |
| --- | --- | --- |
| **推論** | しない（ユーザー入力のみ） | 観測値の入力・追加 |
| **デモ** | する | 生成開始・観測の追加表示・推測照合 |

両モード共通の設定（モード内で常に有効）:

1. **観測系列種別**: 生系列 | 変換系列（`N` 入力）
2. **利用アルゴリズム**: モデル一覧から 1 つ（推論に使用）
3. **逐次推論**: ON | OFF（ON は観測変更ごとに solver 実行、OFF は理論値のみ更新して手動で精密推論）

デモモードのみ追加:

4. **生成源**: モデル実装 | **`Math.random()`（実行環境）**
5. 生成源が `Math.random()` のとき: **利用アルゴリズム**は推論用としてユーザーが明示選択（生成と推論は別設定）

**非仕様**: 生系列と変換系列の同時表示・同時推論・並列比較ワークフロー。

### 3.2 画面レイアウト（完成形）

```
┌──────────────────────────────────────────────────────────────┐
│ ヘッダ                                                        │
├──────────────────────────────────────────────────────────────┤
│ [ 推論 ] [ デモ ]                                             │
├──────────────────────────────────────────────────────────────┤
│ 設定: 観測系列種別 | N | 利用アルゴリズム | 逐次推論 | 生成源  │
├──────────────────────────┬───────────────────────────────────┤
│ メイン                    │ 絞り込み状況（常時・観測≥1）       │

│ デモ: 生成・観測一覧      │  · 推定残り観測数（完全特定まで）  │

├──────────────────────────┴───────────────────────────────────┤

```

## 4. 利用アルゴリズム

各エンジン系統の `Math.random()` 実装を、参照ソースコードに基づく exact モデルとして定義する。UI では系統名を主、実行環境の例を副とする。

| ID | 主表示 | 副表示（例） | 対象 |
| --- | --- | --- | --- |
| `v8` | V8（Node, Chromium） | Node.js, Chrome, Edge | サーバー / ブラウザ |
| `spidermonkey` | SpiderMonkey | Firefox | ブラウザ |
| `javascriptcore` | JavaScriptCore | Safari | ブラウザ |

- 各モードで**利用アルゴリズムは 1 つ**を選択（複数モデルの同時試行は初版では持たない）。
- Node 用 V8 と Chromium 用 V8 が同じモデルで表せる場合は `V8（Node, Chromium）` にまとめる。差分が必要になった場合のみ、`V8（Node）` と `V8（Chromium）` を分ける。
- **バージョン追従・「最新」への更新運用は仕様の対象外**（実装時点のモデルとして固定でよい）。
- モデルは「近似」ではなく、参照した状態遷移・出力変換・出力順序に対する exact モデルとして扱う。
- ただし ECMAScript は `Math.random()` の生成方式を規定しないため、ユーザーの実行環境が選択モデルと一致していることは保証しない。実ブラウザ `Math.random()` を生成源にした場合は、モデル一致 / 不一致を結果として扱う。

### 4.1 モデル説明カード

各モデルには、気になるユーザーが参照元と推論条件を確認できる説明カードを用意する。

| 項目 | 説明 |
| --- | --- |
| モデル名 | UI に表示する名称 |
| 対象環境例 | Node.js, Chrome, Edge, Firefox, Safari など |
| 参照ソース | エンジンのソースコード URL、commit / tag / revision |
| 状態サイズ | 内部状態の bit 数 |
| 出力変換 | 内部状態から `Math.random()` の `Number` への変換方法 |
| 出力順序 | キャッシュ・バッファ・LIFO など、連続観測に影響する順序規則 |
| 出力時に失われる bit | 状態遷移後の値から観測値に出ない bit 数 |
| 必要観測数の目安 | 生系列・変換系列ごとの完全特定に必要な観測数の概算 |
| 既知サンプル | 固定シードや既知系列が用意できる場合の検証ベクトル |

### 4.2 デモモードの生成源

| 生成源 | 推論の利用アルゴリズム |
| --- | --- |
| モデル実装 | 通常は生成源と同じ系統をデフォルト選択。ユーザーが変更可 |
| `Math.random()` | **必ず**ユーザーがモデル一覧から推論用を選択。実行環境の UA 等は参考表示のみ |

生成源 ≠ 利用アルゴリズムのとき、不一致の可能性を絞り込み状況パネルとログに明示する。

## 5. 観測系列種別

### 5.1 切り替え

- ラジオまたはセグメントで **生 | 変換** の一方のみ選択。
- 切り替え時は観測入力をクリアするか確認ダイアログ（仕様: 確認してからクリア推奨）。

### 5.2 生系列

- 入力: `[0, 1)` の実数を表す 10 進文字列。
- 初版では、入力文字列を JS `Number` に parse した結果を**厳密観測値**として扱う。
- これは「10 進の数学的実数を JS が厳密に保持する」という意味ではなく、「観測された JS `Number` と同じ binary64 値を 10 進文字列経由で再入力できる」という意味である。
- `Number.prototype.toString()` などで得られる round-trip 可能な 10 進表記であれば、`Number(text)` で同じ JS `Number` に戻るため、ユーザー入力形式として十分である。
- 丸め・切り捨て・表示桁不足を区間制約として扱う機能は初版対象外とし、将来拡張に回す。

### 5.3 変換系列

- `N`: 2 以上の整数（プリセット例: 6, 10, 100）。
- 入力: `0 .. N-1` の整数のみ。
- 推論モードでは、変換系列を選んだら `N` を必須入力にする。
- 観測値が `N` 以上の場合は入力エラーにする。
- `N` の入力ミスはユーザーの観測条件指定ミスとして扱い、初版では特別な診断対象にしない。

### 5.4 系列種別についての非目標

- 同一セッションで生と変換を並べて比較する画面は作らない。
- 「変換の方が観測が足りない」ことはヘルプと、変換系列選択時の理論目安で伝える。

## 6. 絞り込み状況パネル（リアルタイム）

### 6.1 表示条件

- **推論・デモの両方**で、観測が **1 件以上**あれば常に表示する。
- 観測の追加・削除・編集のたびに更新する。

### 6.2 表示項目

| 項目 | 説明 |
| --- | --- |
| 実推論候補数 | 実際に solver / 推論ロジックで求めた、現在の観測と整合する内部状態の個数 |
| 理論候補数 | 情報量の理論値から概算した候補数。実推論が重い・未完了のときにも表示できる |
| 残り不確実性 | 候補空間の大きさを bit で表した値。例: `残り不確実性: 約 17.2 bit` |
| 信頼度 | 高 / 中 / 低 / 不可（§6.4） |
| 整合観測数 | 全候補で一致する先頭からの件数 |
| 推定残り観測数 | 候補を 1 件に特定するまでの**目安**（完全特定の目標） |
| 推論状態 | `未実行` / `理論のみ` / `推論中` / `候補あり・一意性未確認` / `一意に特定` / `候補複数` / `候補なし` / `タイムアウト` / `solver unavailable` |
| 更新ソース | `推論`（実計算）または `理論`（近似）のラベル |

実推論候補数と理論候補数の両方を出せる場合は併記する。実推論が未完了・タイムアウト・高負荷の場合は、理論候補数を先に表示し、実推論候補数は「計算中」「未実行」「打ち切り」の状態ラベルを出す。

初版では上記を**テキスト・数値**で表示する。理論近似のグラフ・チャート等のビジュアライズは将来拡張とする（§11）。

### 6.3 更新戦略

観測が増えるたびに次のいずれか、または段階的に組み合わせる。

1. **理論近似**: 観測変更時に常に更新する。
   - 1 観測あたりの平均情報量（系列種別・`N`・モデルに依存する係数）
   - 候補空間の対数サイズからの残りビット数
   - 上記から **推定残り観測数** を算出して表示
2. **完全推論**（候補列挙・絞り込み）: 逐次推論が ON の場合は観測変更ごとにデバウンスして実行する。OFF の場合はユーザーが「精密推論を実行」を押したときだけ実行する。
3. 完全推論の結果が得られたら、理論値は上書きせず、実推論値として併記する。表示上は実推論値を主、理論値を参考にする。
4. solver がタイムアウトした場合は、得られる範囲で部分結果を表示する。表示できる候補がある場合は「候補あり・一意性未確認」、一意性確認が終わっていない場合はその旨を明示する。

ユーザー向け文言例:

- 「理論上は ~120 通り程度、実推論では 136 通りに絞り込めています」
- 「残り不確実性: 約 17.2 bit」
- 「完全特定まで、あと約 8 件の観測が必要な見込みです」
- 「推論完了: 候補 3 件（実測）」
- 「候補を 1 件見つけましたが、一意性確認はタイムアウトしました」

### 6.4 信頼度の段階

| 段階 | 目安 |
| --- | --- |
| 高 | 実推論候補数が 1 件（内部状態が 1 通りに定まった状態） |
| 中 | 候補が少数（閾値 T_low 以下、実装で定義） |
| 低 | 候補が多数 |
| 不可 | 入力不正、または選択した利用アルゴリズムと整合する候補 0 |

## 7. 推論モード

### 7.1 入力

| 項目 | 仕様 |
| --- | --- |
| 入力方法 | 貼り付け、カンマ/空白区切り、1 行 1 値、**1 件追加**ボタン |
| 編集操作 | 観測値の個別編集・削除、全クリア、並び順の保持 |
| 利用アルゴリズム | モデル一覧から 1 つ（必須） |
| 観測系列種別 | 生 or 変換 + `N` |
| 逐次推論 | ON / OFF。OFF の場合も理論値は更新し、精密推論は手動実行 |

### 7.2 フロー

1. 観測系列種別・利用アルゴリズムを選択
2. 観測を入力（1 件目から絞り込み状況パネルが更新）
3. 観測を追加・編集・削除するたびにパネルと候補詳細を更新
4. 候補詳細は折りたたみ可能（件数が多いとき）

### 7.3 出力（詳細ペイン）

- 内部状態候補一覧（上限件数・ページング）
- 残り不確実性（bit）
- 曖昧さの理由（観測不足、変換系列、モデル不一致など）
- 次値予測（候補 1 件のときのみ・副）

### 7.4 エラー・警告

- 形式不正、範囲外
- 変換系列で観測値が `N` 以上
- 候補 0（利用アルゴリズム不一致の可能性）
- 連続性の疑い（ヒューリスティック）

## 8. デモモード

### 8.1 設定

| 項目 | 仕様 |
| --- | --- |
| 生成源 | モデル実装 \| `Math.random()` |
| 利用アルゴリズム | モデル一覧から 1 つ（`Math.random()` 生成時は必須・独立選択） |
| 観測系列種別 | 生 or 変換 + `N` |
| 初期状態 | ランダム / シード指定（モデルがサポートする場合） |
| 正解表示 | 生成源がモデル実装の場合、生の内部状態・生の `Math.random()` 値をできるだけ表示 |

### 8.2 フロー

1. 生成源・利用アルゴリズム・観測系列種別を設定
2. 「生成」で系列を作り、観測として先頭から表示
3. **もう 1 件観測に含める**（またはスライダーで観測数 k）ごとに絞り込み状況を更新
4. 詳細ペインにタイムライン: 観測値、変換系列時の元になった生の `Math.random()` 値、候補が 1 のときの予測次値、一致記号
5. 生成を止めたあとも、観測数を変えて絞り込みの変化を試せる
6. 生成源がモデル実装の場合は、正解の内部状態を折りたたみ表示し、推論で得た候補と比較できるようにする

### 8.3 将来オプション

- 生系列の表示桁丸め（推論悪化のデモ）

## 9. 制約と前提

- 観測は単一乱数源からの連続出力。
- `Math.random()` 生成時、推論アルゴリズムはユーザー指定のモデルであり、実行環境との一致は保証しない。
- 変換系列は同じ絞り込み精度に、より多くの観測が必要になりうる（同時比較 UI は提供しない）。
- 完全特定は、内部状態が 1 通りに定まることを指す。次の出力が候補間でたまたま一致しても、内部状態候補が複数ある限り完全特定とはしない。
- 初版では生系列・変換系列ともに厳密観測値のみを扱う。丸め・切り捨て・表示桁不足を考慮した推論は扱わない。
- 推定残り観測数は保証値ではなく、モデル・系列種別に基づく目安。
- 本プロジェクトは `lcg-predictor` と切り離した独立プロジェクトとして扱う。LCG 特化ツールとの機能統合や共通 UI は初版では前提にしない。

## 10. 非機能

| 項目 | 方針 |
| --- | --- |
| リアルタイム性 | 入力デバウンス（例 300ms）、重い推論は Web Worker 化を検討 |
| 表示 | 理論値と実推論値をラベルで区別し、両方ある場合は併記 |
| Solver 不可環境 | Z3 / SharedArrayBuffer 等が使えない場合は、精密推論が利用できない状態として明示する。理論値だけで代替運用することを主目的にはしない |
| プライバシー | 観測値はクライアント内のみ（初版） |

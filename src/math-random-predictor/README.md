# Math.random 推測・検証ツール (Math Random Predictor)

`Math.random()` または `Math.floor(Math.random() * N)` の**連続した観測値**から、内部状態候補を推定し、どれだけ絞り込めているか・完全特定まであと何観測必要かを調べる研究・検証向けプロジェクトです。

現段階では Web UI を先行せず、TDD で core / solver ロジックを固めています。ブラウザ版 UI は、ブラウザ solver の動作確認や結合検証が必要になったタイミングで最小構成から作ります。

## 目的

- 観測された乱数列が、どの程度「続きを推測可能」かを調べる。
- Node.js および主要ブラウザ各系統（V8 / SpiderMonkey / JavaScriptCore）に対応したアルゴリズムモデルを前提に、内部状態候補と一致度を確認する。
- 観測を追加・編集・削除するたびに、候補数・信頼度・推定残り観測数がどう変わるかをリアルタイムで把握する。
- 将来的にはデモモードで生成→推測の流れを再現し、推論モードで外部観測値から同様の分析を行う。

## 目標機能

### 共通

- **観測系列の選択**: 生系列（`Math.random()` 直値）か変換系列（`Math.floor(Math.random() * N)`、`N` 指定）の**どちらか一方**を選んで推論する（同時比較はしない）。
- **利用アルゴリズムの選択**: V8（Node, Chromium）/ Firefox（SpiderMonkey）/ Safari（JavaScriptCore）などのモデルから 1 つを選ぶ。Node 用 V8 と Chromium 用 V8 が同じモデルで表せる場合はまとめて表示する。
- **絞り込み状況パネル**: 観測が 1 件以上ある間、常に表示。候補数・残り不確実性（bit）・信頼度・完全特定までの推定残り観測数などを、観測の追加・編集・削除に連動して更新する（推論が重い場合は情報量の理論値で数値表示。グラフ等のビジュアライズは将来対応）。
- **完全特定の定義**: 内部状態候補が 1 通りに定まること。次の出力が一意に見えるだけでは完全特定とは呼ばない。
- **逐次推論トグル**: ON のときは観測変更ごとに solver による精密推論を走らせる。OFF のときは理論値のみを更新し、ユーザー操作で精密推論を実行する。

### 推論モード（予定）

ユーザーが取得した連続観測値を入力し、選んだアルゴリズムモデルで推定します。

- 生系列: 小数列の貼り付け・手入力・1 件ずつ追加
- 変換系列: 整数列の入力（`N` はユーザー指定）
- 初版では、どちらの系列も厳密観測値のみを扱う。生系列は入力された 10 進文字列を JS `Number` に parse した結果を、観測された `Math.random()` 値そのものとして扱う。

主な表示: 内部状態候補、候補数、残り不確実性（bit）、整合観測数、信頼度、完全特定までの推定残り観測数（次値予測は副）。

### デモモード（予定）

乱数を生成し、同じ UI で推測・照合します。

- **生成源**: 選んだモデル実装、または**実行環境の `Math.random()`**
- `Math.random()` を生成源にした場合: 推論用の利用アルゴリズムをユーザーが**別途**指定する（モデル一覧から選択。実行環境と一致しない可能性を結果に示す）
- 観測系列は推論モードと同様、生系列 / 変換系列のどちらか一方
- 生成源がモデル実装の場合は、正解の内部状態をできるだけ表示する。変換系列を観測対象にしているときも、裏で生成された生の `Math.random()` 値を確認できるようにする。

## 主要モード（予定）

| モード | 概要 |
| --- | --- |
| 推論 | ユーザーが観測値を入力し、選んだアルゴリズムで内部状態候補を推定 |
| デモ | 選んだ生成源で系列を生成し、同じ手順で推測・照合（生成源に `Math.random()` も可） |

各モードで、観測系列の種類（生 / 変換）と利用アルゴリズムを独立に設定できる。

## CLI PoC

UI 実装前の確認用として、Node.js 上で `v8-node-24-cache-lifo-state0` の生成・推論を試せます。

```bash
npm run math-random:cli -- generate --seed 1337 --count 5
npm run math-random:cli -- observe --values "0.9311600617849974 0.3551442693830502 0.7923158995678378 0.7877779424089971" --cache-offset 0
```

現在の CLI は Node.js 24.16.0 の実測列に合わせた `v8-node-24-cache-lifo-state0` モデルを対象にします。

- **生系列**（デフォルト）: GF(2) 線形 solver。`observe --values "0.1 0.2 ..."`
- **変換系列**: Z3 区間制約 solver。`observe --series converted --n 6 --values "5 2 4 4"`

`--cache-offset unknown`（デフォルト）では観測開始位置が cache 内のどこか分からない前提で候補を探索し、`--cache-offset 0..63` を指定すると既知の offset に絞れます。`--max-candidates` は preview 件数の上限です。モデル ID、対象バージョン、実装ソースリンク、Chrome との差分は [ALGORITHM_SURVEY.md](./ALGORITHM_SURVEY.md) に集約しています。

詳細は [SPEC.md](./SPEC.md) を参照してください。

## アルゴリズムの扱い

- 各選択肢は、参照したエンジン実装のソースコードに基づく **exact モデル**として実装する。
- モデルが対象環境の状態遷移・出力変換・出力順序と一致している場合、推論結果はそのモデルに対して exact に扱う。
- 各モデルには、参照ソース、revision、状態サイズ、出力時に失われる bit、必要観測数の目安を確認できる説明カードを用意する。
- Node 用 V8 と Chromium 用 V8 が同じモデルで表せる場合は `V8（Node, Chromium）` のように 1 つにまとめる。差分が必要な場合は別モデルとして分ける。
- ECMAScript は `Math.random()` の生成方式を規定しないため、選択モデルと実行環境が一致しているかは結果表示で明示する。
- デモで `Math.random()` 生成時は、推論アルゴリズムと実行環境の実装が一致しない場合があり、その差を結果として示す。

## 注意事項

- **研究・検証・教育**が主目的。第三者サービスへの攻撃手順の提供を意図しない。
- 観測値は**同一ソースからの連続出力**である必要がある。
- 変換系列は情報量が少ないため、同程度の絞り込みには一般に**より多くの観測**が必要になる（生系列との同時比較機能は持たない）。
- 絞り込みパネルの「残り観測数」は、完全推論が間に合わないとき**理論上の目安**として表示されることがある。
- `Math.floor(Math.random() * N)` の `N` はユーザー入力として必須にし、入力値が `0..N-1` に収まらない場合はエラーにする。
